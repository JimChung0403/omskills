# Legacy VB.NET Analyzer

## 團隊目標

本團隊負責分析舊有 VB.NET WinForms 客戶端應用程式，系統性地探勘專案架構、盤點所有表單、追蹤每個欄位的資料來源與計算邏輯、每個按鈕的動作鏈，並產出結構化的分析文件。

## 通用行為規範

- 使用繁體中文進行所有分析報告的撰寫
- 技術術語保留英文原文，首次出現時附上中文說明
- 所有分析結論必須附上原始碼位置引用（檔案路徑:行號）
- 不得猜測邏輯，所有分析必須基於實際程式碼
- 遇到無法確認的部分，明確標記為「待確認」而非跳過

## 技術約束

- 目標系統：VB.NET WinForms 客戶端應用程式
- 分析工具：使用 Grep、Glob、Read 等工具直接讀取原始碼
- 輸出格式：Markdown
- 程式碼引用格式：`{檔案路徑}:{行號}`

## 部署模式

本團隊使用 **Subagent 模式** 運行。所有 agent 透過 Task tool 由 coordinator 調度，適合循序工作流程與明確的任務交接。

### 使用方式

1. 將 `.claude/` 資料夾部署至目標 VB.NET 專案的根目錄
2. 將本檔案（`CLAUDE.md`）放置於目標專案根目錄
3. 在目標專案根目錄啟動 Claude Code
4. 告知 coordinator 開始分析流程
